# Changelog

Все значимые изменения мозга агента-сисадмина. Формат — по [Keep a Changelog](https://keepachangelog.com/ru/),
версии — по [SemVer](https://semver.org/lang/ru/). Версия мозга — в файле `VERSION`,
релизы помечены git-тегами `vX.Y.Z`.

## [1.3.0] — 2026-05-24

Happ — основной рекомендованный VPN-клиент. Мозг агента приведён в соответствие с
решением курса «Архитектор ИИ-решений» (Модуль 5): основной клиент — **Happ**
(happ.su), остальное (Hiddify, Karing, v2rayN, sing-box, v2rayNG) — альтернативы.
Исправлена несостоятельная аргументация «рекомендуем Hiddify, потому что Happ
удалён из RU App Store, а Hiddify — нет»: на 2026-05-24 в RU App Store нет ни
Happ, ни Hiddify, для iPhone в любом случае нужна смена региона App Store; Happ на
десктоп ставится с happ.su. Формальное обоснование — ADR-0007.

### Добавлено
- **ADR-0007 «Happ — основной рекомендованный VPN-клиент»** — заменяет §5
  «Рекомендации по клиентам» ADR-0005 (остальное ADR-0005 в силе).
- **Append-запись в `_live/timeline.md` (2026-05-24)** — событие «в RU App Store
  нет ни Happ, ни Hiddify; Happ назначен основным клиентом» с источниками
  (наблюдение оператора + curl-проверка happ.su HTTP 200).
- В `client-apps.md` — врезка «основной клиент Happ» (§1) и блок «ключевой вывод
  по RU App Store на 2026-05-24» (§2.2); строка про установку Happ с happ.su (§3.6).

### Изменено
- **Персона §3.8.3** (ядро `sysadmin.md`): рефлекс iPhone-клиента переведён на Happ
  как основной + исправлена аргументация про RU App Store. *(Правка ядра уехала в
  коммит 1.2.0 вместе с inventory-работой — здесь завершён остальной объём VPN-правки.)*
- **`vpn-reflexes.md` §3.8.3** — детальная формулировка: Happ основной, установка
  с happ.su (десктоп) / смена региона (iPhone), явный разбор несостоятельности
  старого аргумента, источник статуса стора.
- **`client-apps.md`** — таблица §2.2 (текущий статус Happ/Hiddify в RU App Store
  + сохранена историческая волна ~28.03.2026), «Универсальный рецепт» (Happ
  основной на всех платформах), врезка on-device в §1 (Happ — основной клиент
  для client-side routing наряду с sing-box; Hiddify по-прежнему не годится).
- **`vpn-consultation-flow.md`** — §2.2 (Happ основной + предупреждение про смену
  региона), §5.4 (TUN: основной пример Happ, подробные шаги оставлены на Hiddify
  как исторически проверенные), §8 (импорт — Happ первым).
- **`frontline-ru.md`** — блок «что работает» и таблица «что выбрать сегодня»: Happ
  основной, статус стора на 2026-05-24 с пометкой источника и confidence.
- **Скилл `generate-client-config`**: `SKILL.md` (финальный отчёт — Happ основной,
  RU App Store на 2026-05-24) и `references/platform-quirks.md` (App Store nuance).
- **ADR-0005 §5** — навигационная пометка «ЗАМЕНЕНО ADR-0007», историческая
  формулировка на 2026-05-15 сохранена.

### Не тронуто (намеренно)
- Hiddify как **технический факт** (sing-box-клиент, формат импорта, версии ядра)
  оставлен везде, где это не «рекомендация по умолчанию».
- Исторические `SESSION-VPN-*.md` и записи timeline до 2026-05-24 — append-only
  след прошлого, задним числом не переписываются.

## [1.2.0] — 2026-05-24

Автоматизации на карте инфраструктуры. По итогам реструктуризации Модуля 6 курса
«Архитектор ИИ-решений»: вместо «панели кнопок» ученик опирается на актуальную
Mermaid-карту от агента как на единый источник правды о том, что крутится само.
Значит, карта обязана отражать автоматизации — раньше они жили только в тексте.

### Добавлено
- **Сбор systemd-таймеров** (`dump-snapshot.sh` → `systemd-timers.txt`): на Ubuntu 24.04
  таймеры — основной способ расписания наравне с cron. Раньше собирались только
  enabled `*.service`, таймеры были полностью невидимы. `set +e` + honest-unknown
  при отсутствии systemctl.
- **Сбор скриптов-наблюдателей** (`watchers.txt`): долгоживущие процессы
  inotify/fswatch/watchdog — отличает «скрипт по расписанию» от «скрипт, слушающий
  событие». На карте это разные роли.
- **Новый inventory-документ `automations.md`** — сводная витрина всех автоматизаций
  независимо от механизма запуска (cron + systemd-timer + watcher + webhook + ручной).
  Колонки `name | trigger | schedule | runs | touches | log | status`. Колонка
  `touches` — источник связей для карты. Тип `automations` добавлен в шаблон
  `inventory-doc-template.md`.
- **Новая mermaid-диаграмма `automations.mmd`** (5-я): триггеры → автоматизации →
  что трогают (БД/сервисы/внешние API). Новый `classDef automation` (фиолетовый
  `#ede9fe/#7c3aed` — не конфликтует с app/db/mon/vpn/infra). Генерируется только
  при наличии автоматизаций (по образцу `vpn-architecture.mmd`).
- **Группа `automations` в `topology.mmd`** — факт наличия + ключевые связи на
  высоком уровне, без детализации триггеров (детали — в `automations.mmd`).
  Появляется только при непустом `automations.md`.

### Изменено
- **Правило синхронности §3.2 персоны** распространено на автоматизации: новая
  cron-задача / systemd-timer / скрипт-наблюдатель / pipeline или изменение того,
  что автоматизация трогает → обновить `automations.mmd` + группу в `topology.mmd`,
  иначе задача не закрыта.
- `inventory-scan/SKILL.md`: description, шаги 2/3/4/4.5/7, Verify-порог snapshot
  (≥14 → ≥16 файлов), 8 → 9 inventory-документов, 4 → 5 диаграмм.
- README + `diagrams/README.md`: состав диаграмм, условие генерации, цветовая семантика.

## [1.1.1] — 2026-05-24

Гибкость менеджеров паролей. По итогам диагностики пользователя на Windows: он
использует Kaspersky Password Manager, которого не было в закрытом enum схемы
(`keychain|bitwarden|1password|pass`) → конфиг невалиден → агент не запускался.
Та же болезнь жёсткого списка, что лечили в 1.1.0 (хардкод вместо гибкости).

### Добавлено
- В enum `secrets.manager`: `keepassxc` (скрипт установки уже был, но в enum отсутствовал)
  и `other` — для любого нестандартного менеджера (Kaspersky, Dashlane, NordPass, браузерный).
- Поля `secrets.manager_name` (имя нестандартного менеджера) и `secrets.cli_available`
  (умеет ли Claude доставать секреты автоматически).
- В `/sysadmin-init`: для `other` агент проводит ресёрч (WebSearch) — есть ли у менеджера
  CLI; честно объясняет расклад (нет CLI → пароли передаёшь руками) и предлагает
  альтернативу Bitwarden. Выбор за оператором. Запрет выдумывать наличие CLI.
- Агент учитывает `cli_available`: при `false` НЕ пытается доставать секреты сам, только
  указывает оператору, где взять руками (cold-start + setup-secrets-vault).

### Изменено
- `validate-config.sh`: jq-fallback принимает 6 менеджеров, требует `manager_name` при `other`.
  WARN про отсутствие check-jsonschema переформулирован в непугающий («необязательно»,
  не «ошибка») — jq-fallback достаточен, check-jsonschema остаётся опциональной надстройкой.

## [1.1.0] — 2026-05-24

Кроссплатформенность установки и принцип «не смог — скажи прямо». По итогам инцидента
с пользователем на Windows: настройка прошла криво, остался мёртвый `infra.md` вместо
папки `infra/` с `sysadmin-config.json` (причина — отсутствие `jq`, который не входит
в Git for Windows, и тихая импровизация агента вместо честной остановки).

### Добавлено
- **Адаптивный гейт окружения** (`_lib/ensure-local-env.sh`): динамический детект
  пакетного менеджера (brew/apt/dnf/yum/zypper/pacman/apk/nix/winget/choco/scoop/port)
  пробегом по PATH вместо привязки «ОС → менеджер». Авто-установка `jq` при отсутствии.
- **Универсальный fallback**: если пакетного менеджера нет — скачивание официального
  бинарника `jq` (github.com/jqlang/jq) под ОС+архитектуру. Работает на любой ОС без менеджера.
- **Финальная самопроверка** (`_lib/self-test-setup.sh`): после настройки агент тестирует,
  что всё реально работает (bash+jq, конфиг валиден, папка `infra/`, bridge-файл). При
  провале — честный вердикт «настройка не завершена, свяжись с разработчиком агента».
- **Конституционное правило C.9** в персоне: запрет на суррогаты конфига/инвентаря
  (никаких самодельных `infra.md`); «не смог → честный вердикт, не видимость работы».

### Исправлено
- Хардкод `/tmp/` заменён на `mktemp -d` (портируемо на Git Bash/WSL/macOS/Linux).
- FINAL CHECK после записи конфига: «Готово» печатается только если файл реально на
  диске и валиден — конец тихим «всё ок» при провалившейся записи.
- `find-config.sh`: `locate_sysadmin_root` понимает Windows-пути в bridge-файле
  (`C:\...`, `C:/...`, `/c/...`) + `cygpath -u`; единый резолвер без дублей grep-regex.
- Авто-добор `~/.sysadmin/bin` в PATH при каждом `source` helper'а — скачанный `jq`
  виден и в новом bash-процессе, и после перезапуска сессии.

### Изменено
- ADR-0002: hard cap ядра персоны поднят 400 → 410 (ради C.9).
- README + INSTALL: требования по окружению (bash обязателен, jq ставится автоматически),
  Windows-специфика. INSTALL клонирует последний тег динамически, без хардкода номера.
