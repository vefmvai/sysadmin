### Категория 6: Shell-скрипты

#### F-6.1 [CRITICAL] — `audit-security/scripts/security-audit.sh:284`

**Где:** `.claude/skills/audit-security/scripts/security-audit.sh:284`
**Что:** `local i=1` внутри `if`-блока на верхнем уровне скрипта (не в
функции). По SC2168 это invalid использование `local`. Bash в strict-mode
печатает warning, в нестрогом — может работать, но поведение не определено
канонически.
**Почему важно:** скилл `audit-security` — read-only диагностический,
оператор будет запускать его регулярно; падение или неожиданное поведение
подорвёт доверие к security-функции.
**Фикс:** убрать `local`, оставить `i=1`. Это обычная переменная в скоупе
скрипта, `local` тут не нужен.
**Группа Phase 2:** C1.

#### F-6.2 [major] — `audit-security/scripts/security-audit.sh:181`

**Что:** `FINDINGS=$(...)` присваивается, нигде далее не используется
(SC2034).
**Фикс:** проверить намерение. Если действительно не используется —
удалить присваивание; если используется неявно (например, через `set -x`
или внешний eval) — добавить `# shellcheck disable=SC2034` с
обоснованием.
**Группа Phase 2:** C1.

#### F-6.3 [major] — `health-check/scripts/full-health.sh:18`

**Что:** `MODE` переменная не используется (SC2034).
**Фикс:** проверить намерение. Скорее всего это плановое расширение «smoke
vs full vs specific»; нужно либо использовать через `case "$MODE"`, либо
удалить, либо добавить `: "${MODE:=full}"` для документации.
**Группа Phase 2:** C1.

#### F-6.4 [major] — `migrate-server-to-server/scripts/02-rsync-incremental.sh:26`

**Что:** `TS=$(date +%Y%m%d-%H%M%S)` не используется (SC2034).
**Фикс:** проверить — возможно, должна включаться в путь LINK_DEST для
incremental snapshot. Если нет — удалить.
**Группа Phase 2:** C1.

#### F-6.5 [major] — `setup-backups/scripts/backup-redis.sh:40`

**Что:** `i` в цикле не используется (SC2034).
**Фикс:** проверить логику цикла. Если `for i in ...; do echo "${j}"; done`
— это bug (вероятно, `j` должен быть `i`). Если намеренно — `for _ in ...`
или disable-комментарий.
**Группа Phase 2:** C1.

#### F-6.6 [major] — `deploy-service/scripts/rollback.sh:56`

**Что:** `<<EOF` без кавычек на маркере — heredoc подвержен expansion на
стороне клиента (SC2087). Это критично для SSH-команд: переменные `$HOME`,
`$USER` и т.д. подставятся локально, а не на удалённом сервере, что
ломает rollback.
**Фикс:** проверить содержимое heredoc. Если переменные должны раскрываться
локально (например, `$PREV_SHA`) — оставить без кавычек, добавить
`# shellcheck disable=SC2087` с обоснованием. Если должны раскрываться
на сервере — `<<'EOF'`.
**Группа Phase 2:** C1.

#### F-6.7..6.13 [style, опционально] — мелкие shellcheck

Семь стилевых замечаний, не влияющих на функциональность. Группа D1 —
опциональная.

- `health-check/scripts/quick-health.sh:228,230` — backticks → `$()`,
  unquoted `${NAME#container-}` (SC2006, SC2086).
- `setup-secrets-vault/scripts/install-pass.sh:39` — `grep|wc -l` →
  `grep -c` (SC2126).
- `inventory-scan/scripts/dump-snapshot.sh:191,203,215` — single-quoted
  heredoc с внутренними single-quotes ломает expansion внутри SSH-вложения
  (SC2016 — нужен disable-комментарий или $()-переход).
- `inventory-scan/scripts/dump-snapshot.sh:227` — `ls` → `find` (SC2012).
- `migrate-server-to-server/scripts/02-rsync-incremental.sh:48,67` —
  `$LINK_DEST` без кавычек (SC2086).
- `audit-security/scripts/security-audit.sh:140` — unquoted variable
  (SC2086).

**Группа Phase 2:** D1 (опционально, может быть отложено).

строк) в новый файл `.claude/agents/references/vpn-reflexes.md`. В ядре
оставить компактный указатель. Если 12 строк недостаточно — также ужать
§3.6 «глубина запроса» (таблица из 8 строк → 4 строки).

**Где:** `.claude/skills/sysadmin-init/SKILL.md`

### Категория 9: Прочие нарушения канона

#### F-9.1 [minor] — `sysadmin-meet/SKILL.md` дубль `disable-model-invocation`

**Где:** `.claude/skills/sysadmin-meet/SKILL.md` (description блок)
**Что:** в frontmatter корректно указано поле `disable-model-invocation:
true`. Параллельно в **тексте** description встречается фраза
«disable-model-invocation: вызывается только явно по /sysadmin-meet...».
Это дубль метаданных в человеческом тексте — путаница между формальным
полем frontmatter и описанием.
**Почему важно:** Claude видит и frontmatter, и description; читатель
может подумать, что это часть текста; формат не canonical.
**Фикс (согласован с Василием):** убрать упоминание имени поля из
description, оставить смысловую формулировку («Вызывается только явно по
/sysadmin-meet или после предложения персоны на однозначные фразы про
запуск процедуры знакомства...»).
**Группа Phase 2:** B1.
