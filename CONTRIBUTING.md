# Вклад в проект

Спасибо, что хочешь улучшить этот проект! Здесь — всё что нужно знать.

## Как сообщить об ошибке

1. Открой [Issues](../../issues) в репозитории.
2. Напиши заголовок — коротко и по сути: `/sysadmin-init падает на macOS Sonoma`.
3. В описании укажи:
   - Что ожидал получить?
   - Что получил? (вывод команды, скриншот)
   - Версия ОС, Docker, shell.
4. Не прикладывай секреты, токены, пароли, IP реальных серверов.

## Как предложить улучшение

1. Форкни репозиторий: `git fork` или кнопка Fork на GitHub.
2. Создай ветку: `git checkout -b feat/название-улучшения`.
3. Внеси изменения, сделай коммит (см. ниже стиль коммитов).
4. Открой Pull Request с описанием: что изменил и зачем.

## Стиль коммитов

Формат: `тип(область): описание на русском`

**Допустимые типы:**

| Тип | Когда использовать |
|-----|-------------------|
| `feat` | Новая функциональность |
| `fix` | Исправление ошибки |
| `docs` | Изменения в документации |
| `chore` | Настройка инструментов, конфигов |
| `refactor` | Рефакторинг без изменения поведения |

**Примеры:**
```
feat(skills): добавлен новый скилл /restore-from-backup
fix(skills): исправлен путь в /sysadmin-init на macOS
docs(readme): уточнено описание /sysadmin-meet
```

## Правило безопасности

**Никогда не коммить:**
- Пароли, токены, API-ключи
- Реальные IP-адреса чужих серверов
- SSH-ключи (приватные)
- `.env`-файлы с реальными значениями

**Рекомендуется настроить gitleaks pre-commit hook** — он автоматически проверяет каждый
коммит на наличие секретов. Установка: `brew install gitleaks`, затем добавь
`gitleaks protect --staged` в `.git/hooks/pre-commit`. Если хук сработал — исправь,
не обходи с `--no-verify`.
