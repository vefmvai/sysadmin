# 0009: stdout/stderr в снимке разделены, ssh-шум заглушён у источника

**Статус:** Принято
**Дата:** 2026-05-25
**Автор:** —

## Контекст

`dump-snapshot.sh` (скилл `/inventory-scan`) собирает снимок сервера, выполняя
команды через SSH и записывая их вывод в файлы снимка. Каждая секция писалась так:

```bash
run_cmd "$cmd" 2>&1 | redact_stream > "$outfile"
```

Прогон у пользователя на Windows/OpenSSH 9.x (отчёт 2026-05-25) показал: в файлы снимка
попадает stderr самого `ssh`-клиента — `mux_client_request_session: ... Connection reset
by peer`, `** WARNING: connection is not using a post-quantum key exchange algorithm`,
`ControlSocket ... already exists, disabling multiplexing`. Особенно при `ControlMaster
auto` в `~/.ssh/config` оператора.

`2>&1` сливает в один поток stderr **двух** источников: удалённой команды (полезная
диагностика — её хотят видеть) и локального ssh-клиента (шум транспорта). После слияния
отличить их невозможно. Ручная чистка `sed`-ом не идемпотентна: следующий скан соберёт
мусор заново.

Рассматривалось:
- **Просто `2>/dev/null`** на весь вызов — убьёт и полезный stderr удалённой команды
  (потеряем «почему секция упала»).
- **Только разделить потоки** (идея из отчёта) — правильное направление, но без рычага №1
  ssh-шум просто переедет в `.stderr.log`, чище не станет.

## Решение

Применяем **два независимых рычага**:

1. **Заглушаем ssh-клиентский шум у источника** — `run_cmd` вызывает
   `ssh -o LogLevel=ERROR -o ControlMaster=no ...`. `LogLevel=ERROR` подавляет
   WARNING/INFO ssh-клиента, но пропускает ERROR/FATAL (реальные сбои видны).
   `ControlMaster=no` отключает мультиплексирование оператора для коротких команд скана
   (источник `mux_client_request_session`).
2. **Разделяем потоки в `run_remote`** — stdout (данные команды) → файл снимка,
   stderr (диагностика удалённой команды) → `<label>.stderr.log` рядом, только если
   непустой. ОБА потока проходят `redact_stream`: stderr тоже может содержать секрет
   (ошибка с connection-string) — маскировка обязательна везде (приоритет №1 CLAUDE.md).

## Последствия

- ✅ Файлы снимка содержат только данные — идемпотентность восстановлена (следующий скан
  не тащит ssh-шум обратно).
- ✅ Диагностика удалённых команд сохранена в `*.stderr.log` (видно, почему секция упала).
- ✅ Кросс-платформенно: лечит Windows/OpenSSH 9.x с `ControlMaster auto`, не завися от
  `~/.ssh/config` оператора.
- ✅ Секреты в stderr маскируются (redaction применяется к обоим потокам).
- ⚠️ Новый класс файлов `*.stderr.log` рядом со снимком. Создаётся только при непустом
  stderr; команды с `2>/dev/null` внутри его не порождают. Verify «≥16 файлов» не ломает.
- ⚠️ Версионирование `.stderr.log` — на усмотрение оператора (снимки в приватной `infra/`,
  не в репозитории мозга).
