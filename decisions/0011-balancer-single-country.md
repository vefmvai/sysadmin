# 0011: Балансир по умолчанию — серверы одной страны (защита от «скачущего IP»)

**Статус:** Принято
**Дата:** 2026-05-26
**Автор:** —

## Контекст

Инцидент 2026-05-26: оператор сообщил, что агент-сисадмин посоветовал его ученику
выстроить VPN-инфраструктуру так, чтобы в панели 3X-UI был балансировщик,
«скачущий» между VPN-серверами разных стран, с утверждением, что это «никак не
влияет на блокировки со стороны Anthropic и ChatGPT».

Факты:

- Утверждение **фактически неверно**. Антифрод нейросетей (OpenAI / Anthropic /
  Google) оценивает **стабильность выходного fingerprint** (IP + ASN + гео + TLS +
  ритм поведения), а не «забанена ли страна». Смена страны внутри сессии = паттерн
  угона аккаунта/ботнета → капчи, ре-логины, запрос верификации телефона, бан.
- Скачущий IP **не обходит** блокировку — он сам **является её триггером**.
- На момент инцидента в проекте уже были защиты от этого антипаттерна, но они не
  закрывали все пути:
  - **Скилл `/configure-vpn-routing`** в граблекейсе запрещал «заведение всех
    серверов подписки в один балансир скопом» — но это инструкция агенту, а не
    барьер; в свободном разговоре (мимо скилла) агент её не применяет.
  - **Код `setup-routing.sh`** принимал `UPSTREAM_TAGS_JSON` и молча собирал
    балансир из тегов любых стран — страну тега скрипт не проверял.
  - **Рефлекса персоны про балансир/стабильность IP не было** — именно через эту
    щель прошёл совет ученику.
- Структурно идентичный риск «Reality на РФ-сервере по инициативе агента» уже был
  закрыт ДВУМЯ слоями: рефлекс персоны 3.8.5 + guard `CONFIRM_REALITY_ON_RU` в
  коде (ADR-0005). Здесь применяется тот же паттерн.
- Рассматривался hard-block (разные страны в балансире запрещены жёстко, обойти
  нельзя). Отклонён: ломает легитимный кейс «самый быстрый пинг любой ценой» для
  НЕ-нейросетевого трафика. Выбран soft-confirm — информированный выбор оператора.

## Решение

Мы делаем **балансир по умолчанию = серверы одной страны** и защищаем это правило
тремя слоями, потому что один слой не закрывает все пути попадания антипаттерна к
пользователю:

1. **Код-guard** в `setup-routing.sh`: при `USE_BALANCER=yes` скрипт проверяет
   `UPSTREAM_COUNTRIES_JSON` (коды стран параллельно тегам). Если стран >1 — или
   список не передан вовсе — без `CONFIRM_MULTI_COUNTRY=yes` делает `exit 3` с
   менторским объяснением. Страну НЕ парсим из имени тега (тег бывает
   `upstream-blanc-usa` / `upstream-server` — угадывание = нарушение правила №1
   CLAUDE.md). Симметрично `CONFIRM_REALITY_ON_RU`.
2. **Рефлекс персоны 3.8.6** (`vpn-reflexes.md` + указатель в `sysadmin.md` §3.8):
   ловит свободный разговор мимо скилла. Балансир из разных стран по своей
   инициативе агент не предлагает; при запросе оператора — менторская развилка
   (объясняю риск → уточняю «это для нейронок?» → при согласии собираю). Фразу
   «балансир между странами не влияет на блокировки» агент никогда не повторяет.
3. **Менторское объяснение в скилле** (`/configure-vpn-routing` Шаг 5A.3 third
   option + усиленный граблекейс + `multi-hop-architectures.md`): пресет «разные
   страны» существует как осознанный выбор с проговорённым риском, а не как
   невозможное или скрытое состояние.

Это **не запрет**, а информированный выбор: легитимный кейс «самый быстрый пинг
любой ценой, НЕ для нейронок» возможен через `CONFIRM_MULTI_COUNTRY=yes`.

## Последствия

- ✅ «Скачущий IP по странам» нельзя получить ни случайно через скилл (guard), ни
  через свободный разговор (рефлекс) — закрыты оба пути, которыми антипаттерн
  доходил до пользователя.
- ✅ Защита симметрична уже существующей паре Reality (3.8.5 + `CONFIRM_REALITY_ON_RU`)
  — единый стиль проекта, предсказуемый для будущих правок.
- ✅ Легитимный multi-hop через разные страны остаётся возможен — осознанно, с
  предупреждением. Свобода оператора не урезана, добавлен только информированный гейт.
- ⚠️ Скилл теперь ОБЯЗАН передавать `UPSTREAM_COUNTRIES_JSON` в `setup-routing.sh`,
  иначе guard сработает (exit 3) даже для серверов одной страны. Это намеренно —
  «нет данных о стране» трактуется как небезопасно.
- ⚠️ Guard не различает два разных «?» (неизвестная страна) — два сервера с
  неопределённой страной пройдут как «одна страна ?». Основной барьер от полного
  отсутствия данных даёт ветка «список не передан → требуем подтверждения».
- ⚠️ Ядро персоны `sysadmin.md` достигло hard cap 410 строк (ADR-0002). Дальнейшие
  рефлексы потребуют либо сжатия, либо пересмотра cap.
