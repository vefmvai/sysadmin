# Sysadmin Agent — твой персональный сисадмин в чате

Готовый агент для Claude Code, который берёт на себя серверную рутину: настройку,
безопасность, бэкапы, мониторинг, деплой, расследования. Ты пишешь нормальными
фразами вроде «проверь сервер» или «настрой бэкапы» — агент сам выбирает нужную
процедуру и выполняет её, объясняя каждый шаг и не делая ничего опасного без твоего
подтверждения.

**Для кого.** Для людей, которые создают IT-продукты с помощью ИИ (вайбкодеров),
имеют один-два собственных сервера и не хотят тратить месяцы на изучение
сисадмин-профессии или нанимать фрилансера за деньги.

**Открытый продукт.** Бери, клонируй, настраивай под
свой проект — всё открыто, ничего не скрыто за облачной подпиской.

---

## Установка — одна команда в Claude Code

Открой Claude Code в любой папке и напиши:

> **Установи агента sysadmin из репо `https://github.com/vefmvai/sysadmin` по инструкции в INSTALL.md.**

Claude всё сделает сам:

1. Клонирует репо себе на машину (спросит куда).
2. Создаст рядом папку `infra/` для твоих данных (карта серверов, ADR, knowledge).
3. Создаст «bridge»-файл, чтобы `@sysadmin` работал из любого проекта.
4. Запустит интервью `/sysadmin-init` — 6 вопросов про твой проект (язык, менеджер паролей, сервер, мониторинг, бэкапы). На выходе — твой личный конфиг.
5. Опционально предложит `/sysadmin-meet` — ~20-минутное знакомство с агентом (рекомендую если впервые видишь).

После этого из **любой папки** в Claude Code можно писать:

```
@sysadmin привет, познакомься с моим сервером
```

Агент сам поймёт что делать, без необходимости запоминать команды или скиллы.

**Системные требования:** macOS / Linux / Windows · `git` · `jq` · Claude Code.

**Обновления.** Агент сам раз в две недели проверит новые версии и предложит обновиться. Или скажи в любой момент «обнови sysadmin» — он подтянет новую версию через `git pull`, покажет changelog и попросит подтверждение перед применением.

**Удаление.** `rm -rf <папка-sysadmin> ~/.claude/agents/sysadmin.md`. Папка `infra/` (твои данные) останется — её удалишь руками если нужно.

## Что внутри

### Агент-персона

Файл `.claude/agents/sysadmin.md` (400 строк ядро) + 7 references-документов в `.claude/agents/references/` — это «мозг» агента: ценности, конституция (что нельзя нарушать ни при каких обстоятельствах), характер, протоколы работы, три зоны безопасности, ритуал памяти, реакции на VPN-сценарии. Всё открыто и читаемо — никакого чёрного ящика. Размер ядра контролируется через ADR-0002 (hard cap 400 строк).

### 18 готовых скиллов

Каждый скилл — это отдельная процедура для типовой задачи. Агент сам выбирает
нужный скилл по твоей фразе.

| Группа | Скиллы | Когда нужно |
|---|---|---|
| **Знакомство и настройка** | `/sysadmin-meet`, `/sysadmin-init` | Первый раз: знакомство → настройка под проект |
| **Настройка с нуля** | `/bootstrap-new-server`, `/setup-secrets-vault`, `/setup-backups`, `/install-monitoring-stack` | После покупки нового VPS |
| **Сеть и обход блокировок** | `/setup-vpn-panel`, `/configure-vpn-routing`, `/setup-server-proxy`, `/generate-client-config`, `/refresh-vpn-knowledge` | Свой VPN-узел, маршрутизация через подписку/свой загр.VPS, серверный прокси для программ, vless-ссылки и QR для устройств, актуализация knowledge раз в месяц (блокировки меняются) |
| **Текущая работа** | `/health-check`, `/inventory-scan`, `/deploy-service` | Регулярно — утренний обход, обновления, диагностика |
| **Безопасность** | `/audit-security`, `/rotate-secrets` | Раз в квартал + при подозрении на проблему |
| **Спецоперации** | `/cleanup-existing-server`, `/migrate-server-to-server` | Унаследованный хаос или переезд на другой провайдер |

Подробное описание каждого — в `/sysadmin-meet` (часть 3) или в `.claude/skills/<имя>/SKILL.md`.

### Inventory — паспорт твоей инфраструктуры

Папка `infra/inventory/` хранит актуальное состояние сервера: контейнеры, домены, базы данных, сети, тома, cron-задачи, автоматизации, TLS-сертификаты + **до 5 mermaid-диаграмм** (общая топология, Docker-сети, маршрутизация доменов, VPN-архитектура, карта автоматизаций). Это не статичная документация — агент сам обновляет её через `/inventory-scan` и при любом архитектурном изменении на сервере.

Inventory **живёт в твоей приватной папке `infra/`**, не в этом публичном репо. У каждого оператора своя инфраструктура и свои данные. Связь между публичным мозгом (этот репо) и твоей `infra/` — через поле `infrastructure.root_path` в `sysadmin-config.json`.

### Конфиг — паспорт твоего проекта

Файл `sysadmin-config.json` живёт **в твоей `infra/`** (не в этом репо). Описывает: язык, менеджер паролей, какие подсистемы включены, какой сервер, секция VPN-блока. Создаётся через `/sysadmin-init`, потом скиллы читают его перед работой.

В этом репо лежат только публичные **шаблон** `sysadmin-config.example.json` и **JSON-схема** `sysadmin-config.schema.json` для валидации.

## Принципы работы

### Три зоны безопасности

Каждое действие на сервере агент классифицирует по одной из трёх категорий:

- **🟢 Зелёная — диагностика.** Что только смотрит, ничего не меняет (логи,
  состояние контейнеров, замер ресурсов). Делает самостоятельно, без вопросов.
- **🟡 Жёлтая — изменения с подтверждением.** Что меняет, но обратимо (запуск
  сервиса, правка конфига). Брифинг из 6 пунктов → пауза → твоё «ок» → выполнение.
- **🔴 Красная — необратимые операции.** Что может уничтожить данные навсегда
  (удаление БД, контейнера с volume). Четырёхшаговая процедура с дословным
  type-to-confirm. Защита от случайностей.

Подробнее — в `/sysadmin-meet` (часть 5) или в персоне (раздел 6 Trust Zones).

### Что агент НЕ делает

- Не пишет твоё приложение (код продукта — твой).
- Не делает бизнес-логику.
- Не работает с серверами в обход своих правил даже под давлением.
- Не делает ничего самовольно, что не входит в текущую задачу.
- Не заменяет твою голову — финальные решения за тобой.

## Требования

- **Claude Code** ([как установить](https://docs.claude.com/en/docs/claude-code/setup))
- **Git**
- **`bash`** — единственное, что нужно обеспечить тебе самому:
  - **macOS / Linux** — `bash` уже встроен, ничего делать не надо.
  - **Windows** — поставь **Git for Windows** (он даёт Git Bash; без него Claude Code
    работает через PowerShell и скиллы агента не запускаются): `winget install --id Git.Git -e`,
    затем перезапусти сессию Claude Code.
- **`jq`** — агент ставит его сам. При первом запуске `/sysadmin-init` проверяет `jq` и, если
  его нет, доустанавливает: сначала через твой пакетный менеджер (brew/apt/dnf/winget/choco/…),
  а если менеджера нет — просто скачивает официальный бинарник `jq` под твою ОС. В конце
  настройки агент сам тестирует, что всё работает, и честно скажет, если что-то не вышло.
- **SSH-доступ** к твоему серверу
- **VPS** (виртуальный сервер) — любой провайдер по выбору. Минимум — 2 ГБ RAM
  если планируешь использовать мониторинг-стек.

> **Менеджер секретов по ОС:** macOS — Keychain (рекомендуется), Linux — pass / KeePassXC /
> Bitwarden, Windows — Bitwarden / KeePassXC. Выбор делается в `/sysadmin-init`.

## Безопасность

- **Секреты НИКОГДА не лежат в репозитории.** Только индекс — в файле
  `inventory/shared/access.md` твоей **приватной** папки `infra/` (создаст
  `/sysadmin-init`). Сами значения — в выбранном менеджере паролей.
- **Pre-commit hook** через gitleaks (рекомендуется настроить — см. `CONTRIBUTING.md`).
  Ловит случайные коммиты секретов.
- **Конфиг проекта** (`sysadmin-config.json`) — в `.gitignore`, не попадает в публичный репо.
- **Конституция агента** запрещает выдумывать данные, обходить процедуры, делать
  деструктивы без явного подтверждения.

## Углубление

Когда захочешь разобраться глубже — карта в `/sysadmin-meet` (часть 7). Краткий
порядок чтения:

1. **`CLAUDE.md`** — главный документ проекта (обязательно в первую неделю)
2. **`.claude/agents/sysadmin.md`** — полный мозг агента
3. **`.claude/skills/<имя>/SKILL.md`** — конкретный скилл, который тебя интересует
4. **`decisions/0000-template.md`** — шаблон ADR. Когда будешь принимать своё
   архитектурное решение — копируй шаблон в свою `infra/decisions/` и заполняй.
