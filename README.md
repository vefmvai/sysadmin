# Sysadmin Agent — твой персональный сисадмин в чате

Готовый агент для Claude Code, который берёт на себя серверную рутину: настройку,
безопасность, бэкапы, мониторинг, деплой, расследования. Ты пишешь нормальными
фразами вроде «проверь сервер» или «настрой бэкапы» — агент сам выбирает нужную
процедуру и выполняет её, объясняя каждый шаг и не делая ничего опасного без твоего
подтверждения.

**Для кого.** Для людей, которые создают IT-продукты с помощью ИИ (вайбкодеров),
имеют один-два собственных сервера и не хотят тратить месяцы на изучение
сисадмин-профессии или нанимать фрилансера за деньги.

**Открытый продукт.** Бери, клонируй, настраивай под
свой проект — всё открыто, ничего не скрыто за облачной подпиской.

---

## С чего начать (3 простых шага)

### Шаг 1: Клонировать репозиторий

```bash
git clone <repo-url> sysadmin
cd sysadmin
```

### Шаг 2: Открыть Claude Code и запустить знакомство

```
/sysadmin-meet
```

Это **подробное введение на простом языке** (~20 минут вдумчивого чтения, 7 частей
с паузами). Узнаешь:

- Что такое агент и какую боль решает
- Базовый словарь (8 ключевых слов с бытовыми аналогиями)
- Что умеют все 17 скиллов агента
- Как с агентом разговаривать (с примерами реальных диалогов)
- Принципы безопасности — три зоны
- Где границы и куда смотреть дальше за углублением

Если уже знаком с агентом и хочешь сразу настроить — пропускай знакомство и иди
к Шагу 3.

### Шаг 3: Настроить агента под твой проект

```
/sysadmin-init
```

Интерактивная настройка: 6 вопросов про твой проект (язык, менеджер паролей,
сервер, мониторинг, бэкапы, Telegram-уведомления). На выходе — файл
`sysadmin-config.json`. Занимает ~5 минут.

После этого можно начинать реальную работу:

```
@sysadmin привет, познакомься с моим сервером
```

## Что внутри

### Агент-персона

Файл `.claude/agents/sysadmin.md` (~1100 строк) — это «мозг» агента: ценности,
конституция (что нельзя нарушать ни при каких обстоятельствах), характер, протоколы
работы, три зоны безопасности, ритуал памяти. Всё открыто и читаемо — никакого
чёрного ящика.

### 17 готовых скиллов

Каждый скилл — это отдельная процедура для типовой задачи. Агент сам выбирает
нужный скилл по твоей фразе.

| Группа | Скиллы | Когда нужно |
|---|---|---|
| **Знакомство и настройка** | `/sysadmin-meet`, `/sysadmin-init` | Первый раз: знакомство → настройка под проект |
| **Настройка с нуля** | `/bootstrap-new-server`, `/setup-secrets-vault`, `/setup-backups`, `/install-monitoring-stack` | После покупки нового VPS |
| **Сеть и обход блокировок** | `/setup-vpn-panel`, `/configure-vpn-routing`, `/setup-server-proxy`, `/generate-client-config` | Свой VPN-узел, маршрутизация через подписку/свой загр.VPS, серверный прокси для программ, vless-ссылки и QR для устройств |
| **Текущая работа** | `/health-check`, `/inventory-scan`, `/deploy-service` | Регулярно — утренний обход, обновления, диагностика |
| **Безопасность** | `/audit-security`, `/rotate-secrets` | Раз в квартал + при подозрении на проблему |
| **Спецоперации** | `/cleanup-existing-server`, `/migrate-server-to-server` | Унаследованный хаос или переезд на другой провайдер |

Подробное описание каждого — в `/sysadmin-meet` (часть 3) или в `.claude/skills/<имя>/SKILL.md`.

### Inventory — паспорт твоей инфраструктуры

Папка `inventory/` хранит актуальное состояние сервера: контейнеры, домены, базы
данных, сети, тома, cron-задачи, TLS-сертификаты. Это не статичная документация —
агент сам обновляет её через `/inventory-scan`. Без актуального inventory агент
работает вслепую, поэтому он сам отслеживает свежесть и просит обновить если данным
больше месяца.

Inventory **не лежит в этом репо** — у каждого оператора своя инфраструктура и свои
данные. Папка `inventory/` появится в твоей **отдельной** папке `infra/` (которую
создаст `/sysadmin-init`). Связь между этим репо и твоей `infra/` — через поле
`infrastructure.root_path` в `sysadmin-config.json`.

### Конфиг — паспорт твоего проекта

Файл `sysadmin-config.json` в корне (личный, в `.gitignore`) описывает твой
конкретный проект: язык, менеджер паролей, какие подсистемы включены, какой сервер.
Создаётся через `/sysadmin-init`, потом несколько скиллов читают его перед работой.

Публичный шаблон `sysadmin-config.example.json` + JSON-схема
`sysadmin-config.schema.json` для валидации.

## Принципы работы

### Три зоны безопасности

Каждое действие на сервере агент классифицирует по одной из трёх категорий:

- **🟢 Зелёная — диагностика.** Что только смотрит, ничего не меняет (логи,
  состояние контейнеров, замер ресурсов). Делает самостоятельно, без вопросов.
- **🟡 Жёлтая — изменения с подтверждением.** Что меняет, но обратимо (запуск
  сервиса, правка конфига). Брифинг из 6 пунктов → пауза → твоё «ок» → выполнение.
- **🔴 Красная — необратимые операции.** Что может уничтожить данные навсегда
  (удаление БД, контейнера с volume). Четырёхшаговая процедура с дословным
  type-to-confirm. Защита от случайностей.

Подробнее — в `/sysadmin-meet` (часть 5) или в персоне (раздел 6 Trust Zones).

### Что агент НЕ делает

- Не пишет твоё приложение (код продукта — твой).
- Не делает бизнес-логику.
- Не работает с серверами в обход своих правил даже под давлением.
- Не делает ничего самовольно, что не входит в текущую задачу.
- Не заменяет твою голову — финальные решения за тобой.

> одинаковый у всех, кто его берёт. Приватный `infra/` — твои реальные данные,
> которые публиковать нельзя. Связь — через `infrastructure.root_path` в `sysadmin-config.json`.

## Требования

- **Claude Code** ([как установить](https://docs.claude.com/en/docs/claude-code/setup))
- **Git**
- **macOS или Linux** (агент по умолчанию рекомендует Keychain для секретов на macOS,
  pass / KeePassXC / Bitwarden — на Linux)
- **SSH-доступ** к твоему серверу
- **VPS** (виртуальный сервер) — любой провайдер по выбору. Минимум — 2 ГБ RAM
  если планируешь использовать мониторинг-стек.

## Безопасность

- **Секреты НИКОГДА не лежат в репозитории.** Только индекс — в файле
  `inventory/shared/access.md` твоей **приватной** папки `infra/` (создаст
  `/sysadmin-init`). Сами значения — в выбранном менеджере паролей.
- **Pre-commit hook** через gitleaks (рекомендуется настроить — см. `CONTRIBUTING.md`).
  Ловит случайные коммиты секретов.
- **Конфиг проекта** (`sysadmin-config.json`) — в `.gitignore`, не попадает в публичный репо.
- **Конституция агента** запрещает выдумывать данные, обходить процедуры, делать
  деструктивы без явного подтверждения.

## Углубление

Когда захочешь разобраться глубже — карта в `/sysadmin-meet` (часть 7). Краткий
порядок чтения:

1. **`CLAUDE.md`** — главный документ проекта (обязательно в первую неделю)
2. **`.claude/agents/sysadmin.md`** — полный мозг агента
3. **`.claude/skills/<имя>/SKILL.md`** — конкретный скилл, который тебя интересует
4. **`decisions/0000-template.md`** — шаблон ADR. Когда будешь принимать своё
   архитектурное решение — копируй шаблон в свою `infra/decisions/` и заполняй.
