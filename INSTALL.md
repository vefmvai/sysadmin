# INSTALL.md — инструкция установки sysadmin для Claude Code

> **Этот файл предназначен для чтения Claude Code, не для человека.** Если ты человек и хочешь поставить агента — открой Claude Code в любой папке и напиши:
>
> > Установи агента sysadmin из репо `https://github.com/vefmvai/sysadmin` по инструкции в INSTALL.md.
>
> Claude прочитает этот файл и сделает всё сам. Описание для людей — в `README.md`.

---

## Что ставим

Агент-сисадмин для Claude Code. Архитектурно это:

- **Публичный репозиторий `sysadmin/`** — мозг агента: персона, 18 скиллов, knowledge-база, ADR. Обновляется через `git pull` из этого репо. Версионирован git-тегами (`v1.0.0`, `v1.1.0`, …).
- **Приватная папка оператора `infra/`** — его данные: карта серверов (inventory), ADR по его инфре, инциденты, knowledge про его сервер, личный `sysadmin-config.json`. **Создаётся при установке**, дальше живёт у оператора, не публикуется.
- **Bridge-файл `~/.claude/agents/sysadmin.md`** — маленький указатель, который говорит Claude: «есть subagent `sysadmin`, мозг живёт по такому-то пути». Благодаря этому файлу `@sysadmin` доступен из любой папки.

## Что должен сделать Claude по этой инструкции

Девять шагов. Между шагами 4 и 5 — два вопроса оператору. Остальное — автоматически.

### Шаг 1. Pre-check окружения

Проверь наличие:
- `git` — `command -v git`. Если нет — сообщи оператору как установить под его ОС, остановись.
- `jq` — `command -v jq`. Если нет — сообщи как установить (`brew install jq` / `apt-get install jq` / `winget install --id jqlang.jq -e`), остановись.
- Claude Code установлен — раз оператор читает это, очевидно да.

Определи ОС (`uname -s`): `Darwin`, `Linux`, `MINGW*` / `MSYS*` / `CYGWIN*` для Windows. Это нужно для генерации правильного пути в bridge-файле (Шаг 7).

**Windows-специфика (важно — частая причина кривой установки):**
- Если ты читаешь это, bash уже работает (раз команды исполняются). Но на нативном Windows
  Claude Code использует bash **только если установлен Git for Windows**; без него команды идут
  через PowerShell и установка не пройдёт. Симптом — команды «молчат» или дают PowerShell-ошибки.
  Реакция: STOP, попроси поставить Git for Windows (`winget install --id Git.Git -e`) + перезапуск сессии.
- `jq` **не входит** в Git for Windows. После `winget install --id jqlang.jq -e` пакет ставится,
  но в текущей сессии может ещё не появиться в PATH — попроси оператора **перезапустить сессию**
  Claude Code и запустить установку заново. NEVER продолжать установку без работающего `jq` —
  иначе родится мёртвый суррогат вместо нормальной папки `infra/` (см. C.9 персоны, инцидент 2026-05-24).

### Шаг 2. Проверить нет ли уже установленного sysadmin

Проверь `~/.claude/agents/sysadmin.md`:
- **Нет** — нормально, продолжаем установку.
- **Есть** — это **повторная установка** или **переустановка**. Спроси оператора:
  - Если он впервые ставит и в файле что-то странное — переименуй существующий в `sysadmin.md.bak.YYYYMMDD-HHMMSS` (с timestamp), сообщи что сделал, продолжай.
  - Если он сознательно переустанавливает — спроси: «У тебя уже есть bridge-файл агента. Сохранить старый как backup и поставить новый? `[Сохранить старый и продолжить] / [Прервать]`».

### Шаг 3. Спросить — куда клонировать публичный репо sysadmin

Используй `AskUserQuestion` (radio):

> Куда склонировать публичный репо sysadmin (мозг агента)?
>
> 1. **`~/sysadmin`** (рекомендуется) — простой, понятный путь.
> 2. **`~/work/sysadmin`** — если у тебя проекты в `~/work/`.
> 3. **`~/projects/sysadmin`** — если у тебя проекты в `~/projects/`.
> 4. **Другой путь** — оператор укажет вручную.

Сохрани результат как `$SYSADMIN_PATH` (абсолютный путь, tilde раскрыта). Проверь что:
- Родитель этого пути существует (`dirname "$SYSADMIN_PATH"` — `[ -d ]`).
- Сам путь **не существует** (мы будем клонировать в эту папку). Если существует — спроси: «По этому пути уже что-то есть. Использовать его как клон sysadmin? `[Использовать] / [Выбрать другой путь]`».

### Шаг 4. Спросить — куда положить приватную папку infra

Используй `AskUserQuestion` (radio):

> Куда положить приватную папку `infra/` (карта твоих серверов, ADR, knowledge)?
> Рекомендация — рядом с `sysadmin/`, чтобы они жили парой.
>
> 1. **`<родитель sysadmin>/infra`** (рекомендуется) — соседняя папка.
> 2. **`~/infra`** — отдельно, в home.
> 3. **Другой путь** — оператор укажет.
>
> Эта папка приватная — не публикуй её. Если у тебя уже есть git-репо с инфрой — укажи путь к нему, я с ним подружу sysadmin (не буду переписывать существующее).

Сохрани как `$INFRA_PATH`. Если путь **существует и содержит git-репо** — это сценарий «у меня уже есть infra/, подключи к ней sysadmin». Если **не существует** — создадим в Шаге 6.

### Шаг 5. Сообщить план оператору и получить подтверждение

Выведи 6-пунктовый брифинг (Yellow Zone — мы пишем файлы):

```
План установки sysadmin:

1. ЧТО ДЕЛАЮ:
   - git clone https://github.com/vefmvai/sysadmin <SYSADMIN_PATH>
   - mkdir -p <INFRA_PATH> (если папки нет)
   - создаю ~/.claude/agents/sysadmin.md с указателем на мозг
   - запускаю /sysadmin-init для интервью (язык, менеджер паролей, сервер)
   - предлагаю /sysadmin-meet для знакомства (~20 мин, опционально)

2. ЗАЧЕМ: чтобы агент @sysadmin был доступен из любой папки Claude Code,
   с автоматической маршрутизацией задач про серверы/Docker/VPN/бэкапы.

3. ЧТО ПРОИЗОЙДЁТ: создам 2 папки, 1 bridge-файл, проведу интервью на 5 минут.
   Существующие файлы не трогаю без подтверждения.

4. ЧТО ПРОВЕРИЛ: git и jq установлены; путей <SYSADMIN_PATH>/<INFRA_PATH>
   нет в коллизиях с существующими файлами.

5. РИСК: низкий. Откат: rm -rf <SYSADMIN_PATH> <INFRA_PATH> + rm ~/.claude/agents/sysadmin.md.

6. СТРАХОВКА: каждый шаг логирую в чат. Если что-то не так — оператор скажет «стоп».

Поехали? (да / нет / задать вопрос)
```

Без явного «да» / «давай» / «ok» — **не продолжай**.

### Шаг 6. Клонировать репо и создать infra

```bash
# Клонируем последний релизный тег (а не main), для стабильности.
# Это даёт detached HEAD на теге — правильное состояние для версионированной
# установки: tree точно соответствует тегу, случайные коммиты невозможны.
# Обновление через команду «обнови sysadmin» выполняет git fetch + git checkout --detach
# на новый тег, сохраняя ту же модель работы.
git clone --branch v1.0.0 https://github.com/vefmvai/sysadmin "$SYSADMIN_PATH"

# Если v1.0.0 не существует (репо ещё без тегов) — fallback на main с предупреждением
if [ $? -ne 0 ]; then
    echo "WARN: тег v1.0.0 не найден, клонирую main (нестабильно)"
    git clone https://github.com/vefmvai/sysadmin "$SYSADMIN_PATH"
fi

# Создаём infra/ если её нет
if [ ! -d "$INFRA_PATH" ]; then
    mkdir -p "$INFRA_PATH"/{inventory,decisions,incidents,knowledge,runbooks}
    cd "$INFRA_PATH" && git init -q && cd -
    cat > "$INFRA_PATH/README.md" <<'INFRA_README'
# infra — приватная инфра-папка оператора

Эта папка — твои данные о серверах: inventory (карта), decisions (ADR), knowledge
(уроки), incidents (инциденты), runbooks (процедуры). Связана с публичным мозгом
агента sysadmin через ~/.claude/agents/sysadmin.md.

**Не публикуй** эту папку — здесь личные данные. Если хочешь версионировать —
создай приватный git-репо.

Структура:
- inventory/   — карта серверов и сервисов (обновляется через `/inventory-scan`)
- decisions/   — ADR по архитектуре твоей инфры
- knowledge/   — что выстрадано на твоём сервере (lessons-learned и т.п.)
- incidents/   — постмортемы инцидентов
- runbooks/    — пошаговые процедуры под твою инфру

`sysadmin-config.json` появится тут после `/sysadmin-init` (Шаг 8).
INFRA_README
fi
```

### Шаг 7. Создать bridge-файл `~/.claude/agents/sysadmin.md`

Это ключевой файл, который делает `@sysadmin` доступным из любой папки. Если оператор использует Yandex.Disk / Dropbox / iCloud для синхронизации `$SYSADMIN_PATH` — это нужно отметить в bridge (предупреждение про синхронизацию облака).

```bash
mkdir -p ~/.claude/agents

# Если файл существует и не было подтверждения замены в Шаге 2 — стоп
if [ -f ~/.claude/agents/sysadmin.md ] && [ "$REPLACE_CONFIRMED" != "yes" ]; then
    echo "STOP: bridge уже есть, не было подтверждения замены"
    exit 1
fi

# Бэкап если был
[ -f ~/.claude/agents/sysadmin.md ] && \
    mv ~/.claude/agents/sysadmin.md ~/.claude/agents/sysadmin.md.bak.$(date +%Y%m%d-%H%M%S)

# Проверяем, не лежит ли $SYSADMIN_PATH в облачной папке
CLOUD_WARNING=""
case "$SYSADMIN_PATH" in
    *Yandex.Disk*|*Dropbox*|*iCloud*|*OneDrive*|*Google\ Drive*|*Mega*)
        CLOUD_WARNING="Внимание: sysadmin/ хранится в облачной синхронизации. Файлы могут расходиться между машинами при одновременной работе." ;;
esac

cat > ~/.claude/agents/sysadmin.md <<BRIDGE_EOF
---
name: sysadmin
description: >
  Агент-сисадмин для управления серверной инфраструктурой.
  Вызывай для: инфраструктурных задач, диагностики сервера, управления Docker,
  работы с PostgreSQL/Redis/MySQL, проверки TLS-сертификатов и nginx, любых SSH-операций,
  настройки VPN-панели 3X-UI, серверного прокси, генерации клиентских VPN-конфигов,
  бэкапов, мониторинга, ротации секретов, аудита безопасности, миграции серверов.
  Также — команда «обнови sysadmin» (проверка/применение обновлений мозга через git pull).
  Использует политику трёх зон доверия (зелёная/жёлтая/красная) и сеньор-ментор стиль —
  объясняет, обучает, оспаривает опасные решения. Все скиллы выбирает сам — оператор
  просто описывает задачу словами, не запоминая имена команд.
tools: Bash, Read, Write, Edit, Glob, Grep
model: inherit
---

# Sysadmin Agent — bridge

Я — агент-сисадмин. Полный мозг (персона, конституция, 18 скиллов, knowledge) живёт:

**\`$SYSADMIN_PATH\`**

## Cold Start

1. Прочитай ядро персоны: \`$SYSADMIN_PATH/.claude/agents/sysadmin.md\`
2. Дальше следуй Cold Start Protocol из персоны (\`references/cold-start.md\`):
   - Найди \`sysadmin-config.json\` в \`infra/\` оператора по стандартному алгоритму
   - Прочитай \`infra/inventory/README.md\` и \`infra/inventory/topology.md\`
   - Прочитай долговременную память: \`infra/knowledge/lessons-learned.md\`, \`operator-profile.md\`, \`working-patterns.md\`
3. ${CLOUD_WARNING:+Предупреди оператора: $CLOUD_WARNING При работе попроси подтвердить актуальность inventory.}
4. Только после этого берись за задачу.

## Скиллы и knowledge

В \`$SYSADMIN_PATH/.claude/skills/\` — 18 готовых скиллов. В \`.claude/knowledge/\` — доменные знания.
Я выбираю и применяю их сам по формулировке задачи оператора. Оператор не должен запоминать имена.

## Если что-то не нашёл

Если \`$SYSADMIN_PATH/.claude/agents/sysadmin.md\` не существует — репо переехал или удалён. Сообщи оператору и попроси указать актуальный путь.
BRIDGE_EOF

echo "Создан bridge: ~/.claude/agents/sysadmin.md → $SYSADMIN_PATH"
```

### Шаг 8. Запустить интервью /sysadmin-init

Перейди в `$SYSADMIN_PATH` (важно — без этого скилл не подхватится) и попроси оператора:

> Сейчас запущу интервью /sysadmin-init — 5 минут, 6 вопросов. На выходе появится `$INFRA_PATH/sysadmin-config.json`.
>
> Вводи команду:
>
>     /sysadmin-init
>
> Когда интервью закончится — вернись сюда, я предложу следующий шаг.

(Claude не запускает скилл напрямую — оператор вводит `/sysadmin-init` сам. Скилл сделает остальное и попросит указать путь к infra — `$INFRA_PATH`.)

### Шаг 9. Финал — предложить /sysadmin-meet и завершить установку

После того как оператор прошёл `/sysadmin-init`:

> Готово. Установка завершена. У тебя теперь:
>
> - **Bridge**: `~/.claude/agents/sysadmin.md` (Claude видит @sysadmin везде)
> - **Мозг**: `$SYSADMIN_PATH` (публичный, обновляется через `git pull`)
> - **Данные**: `$INFRA_PATH` (приватный, твой)
> - **Конфиг**: `$INFRA_PATH/sysadmin-config.json` (заполнен по твоим ответам)
>
> **Что дальше:**
>
> 1. **Если первый раз видишь агента** — рекомендую `/sysadmin-meet` (~20 мин, ELI5-знакомство). Можно запустить прямо сейчас.
> 2. **Иначе** — просто из любой папки пиши `@sysadmin привет, познакомься с моим сервером` — он сам разберётся.
>
> **Обновления:** агент сам раз в 14 дней проверит новые версии и предложит обновиться. Команда «обнови sysadmin» — в любой момент.
>
> **Удаление:** `rm -rf $SYSADMIN_PATH ~/.claude/agents/sysadmin.md`. Папка `$INFRA_PATH` (твои данные) останется.

## Что НЕ делать (anti-patterns для Claude)

- **Не клонировать в `/tmp` или системные папки** — папка живёт долго, должна быть в home.
- **Не запускать `/sysadmin-init` без `cd $SYSADMIN_PATH`** — скилл не подхватится из чужой папки.
- **Не молча перезаписывать существующий bridge или infra/** — всегда подтверждение, backup при замене.
- **Не делать `git pull` на main без тега** — тег `v1.0.0` (или последний) обязателен для стабильности.
- **Не пытаться поставить агента на сам сервер** — sysadmin живёт на машине оператора (ноутбуке), управляет сервером через SSH. Установка **на сервер** — нонсенс.
- **Не делать `chmod +x` на скрипты `infra/scripts/`** — это работа `/bootstrap-new-server` на сервере, не локально.

## Если оператор просит «поставь без интервью» / «без вопросов»

Минимальная установка возможна, но **`/sysadmin-init` всё равно нужно запустить** — иначе агент не знает оператора, его язык, менеджер паролей, сервер. Если оператор настаивает:

> Я могу сделать клонирование и bridge без интервью, но без `sysadmin-config.json` агент не сможет работать полноценно — он будет постоянно просить запустить `/sysadmin-init`. Точно пропустить интервью?

При повторном «да» — пропусти Шаги 8-9, оставь установку наполовину. Оператор сам запустит `/sysadmin-init` когда будет готов.
