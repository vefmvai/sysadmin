# Evals для /configure-vpn-routing

> Источник правды для `description` в SKILL.md. Сначала пишутся triggers,
> потом из них формулируется description. При расхождении — описание
> обновляется под triggers.

## ДОЛЖНО триггерить (positive — минимум 3)

- «настрой маршрутизацию VPN»
- «добавь клиента VPN»
- «настрой VLESS на установленной панели»
- «семье нужен VPN — добавь 3 человек»
- «теперь хочу подключить телефон»
- «настрой outbound через подписку»
- «вот моя подписная ссылка от провайдера, добавь её как outbound»
- «добавь второй сервер в balancer»
- «нужен мульти-хоп через мой загр.VPS»
- «настрой РФ напрямую, остальное через VPN»
- «добавь outbound через свой загр.сервер»
- «configure-routing»

## НЕ ДОЛЖНО триггерить (negative — минимум 3)

- «настрой VPN-панель на сервере» — это `/setup-vpn-panel` (установка панели с нуля)
- «поставь 3X-UI» — это `/setup-vpn-panel`
- «настрой прокси для бота на сервере» — это `/setup-server-proxy` (mixed inbound для программ)
- «сделай VPN-конфиг для iPhone из подписки» — это `/generate-client-config` (генерация клиентских артефактов)
- «удали клиента alice» — обратная операция, не покрывается этим скиллом (можно через прямой API-вызов)
- «обнови 3X-UI до новой версии» — это `/deploy-service`, не этот скилл
- «расскажи про balancer» — это вопрос про знания, ответ персоны со ссылкой на multi-hop-architectures.md
- «как работает Reality» — вопрос про знания, ответ персоны со ссылкой на vpn-protocols.md
- «настрой VPN на ноутбуке» — это клиентское приложение, не серверная маршрутизация

## Edge-cases

- «настрой VPN» (без контекста) — должен уточнить «панель уже установлена или нет?». Если нет → `/setup-vpn-panel` сначала. Если да → этот скилл.
- «подключи провайдера X» — должен уточнить «у тебя есть подписная ссылка или прямой vless://-конфиг?». Если есть → этот скилл с `OUTBOUND_KIND=subscription`. Если нет → персона объясняет про получение подписки у провайдера.
- «настрой outbound на свой загр.VPS» — должен уточнить «на загр.VPS уже создан VLESS+Reality inbound?». Если нет → сначала `/setup-vpn-panel --location=foreign-server` на загр.VPS, потом `/configure-vpn-routing --inbound-only` там же, потом сюда возвращаемся.
- «семья жалуется что не работает Hulu» — это не настройка VPN, это сначала troubleshooting. Этот скилл активируется только когда оператор явно просит изменить настройки.
- «добавь клиента и сразу пришли мне QR» — два шага: этот скилл (добавление клиента) + `/generate-client-config` (QR). Скилл может предложить chain.
