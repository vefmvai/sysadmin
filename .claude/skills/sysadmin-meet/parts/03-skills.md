**Часть 3 из 7. Что агент умеет — все 17 скиллов с человеческими описаниями**

У агента 17 скиллов. Я разбил их на 5 групп по смыслу — так легче запомнить.

**Группа 1. Настройка с нуля (запускаются один раз, в начале).**

**`/sysadmin-init`** — Настройка агента под твой проект. Создаёт паспорт проекта
(`sysadmin-config.json`). В конце знакомства я тебя сюда отправлю.
*Когда нужно:* самый первый шаг после знакомства. И повторно, если что-то поменял
(новый сервер, другой менеджер паролей).

**`/bootstrap-new-server`** — Настройка свежего сервера с нуля. Когда ты только
купил VPS у провайдера, это голая Ubuntu без ничего. Этот скилл за один проход
делает всё что нужно для безопасной базы: настраивает SSH-доступ, ставит файрвол,
ставит Docker, готовит структуру папок. После него сервер готов принимать твои
приложения.
*Когда нужно:* один раз, сразу после покупки VPS.

**`/setup-secrets-vault`** — Настройка менеджера паролей. Помогает выбрать и
настроить безопасное хранилище для твоих токенов и паролей (Keychain на маке,
KeePassXC, Bitwarden, pass на Linux). Это нужно, потому что главное правило агента:
**секреты НИКОГДА не лежат в репозитории**, только в защищённом хранилище.
*Когда нужно:* один раз, в начале работы с проектом.

**`/setup-backups`** — Настройка бэкапов. Поднимает на сервере систему
автоматического резервного копирования: каждый день делает снимок твоих баз данных,
шифрует и отправляет в облачное хранилище (Яндекс.Диск, S3, B2 — на выбор). Также
настраивает алерт «бэкап старше 36 часов — что-то сломалось».
*Когда нужно:* один раз, **до того** как начнёшь делать что-то важное на сервере.

**`/install-monitoring-stack`** — Установка системы мониторинга. Поднимает на твоём
сервере дашборд с глазами: Uptime-Kuma (показывает живы ли сайты), Beszel (графики
CPU/памяти/диска), Dozzle (логи в браузере), Dockge (управление контейнерами через
веб-интерфейс), Diun (уведомления когда выходят обновления Docker-образов). Всё это
доступно через один поддомен с защитой по паролю. Алерты идут в Telegram.
*Когда нужно:* опционально, для production-серверов. Можно жить и без него, но
неудобно.

**Группа 2. Текущая работа (используется регулярно).**

**`/health-check`** — Быстрая проверка здоровья сервера. За 10 секунд показывает:
живы ли все контейнеры, не кончилось ли место на диске, свежий ли бэкап, валидны
ли SSL-сертификаты. Безопасный — ничего не меняет, только смотрит.
*Когда нужно:* утренний обход, перед тем как что-то делать, при подозрении что
что-то не так.

**`/inventory-scan`** — Обновление инвентаря. Лезет на сервер, снимает свежий
снимок (что запущено, какие сети, какие домены, какие сертификаты) и обновляет файлы
в папке `inventory/`. Безопасный — только читает, ничего не меняет.
*Когда нужно:* после любых крупных изменений, или раз в месяц для актуализации.

**`/deploy-service`** — Деплой нового сервиса или обновление существующего. Через
этот скилл ты добавляешь на сервер новые контейнеры (новый сайт, новый бот) или
выкатываешь обновления. Со встроенным rollback — если что-то пошло не так, откат
за 5 минут.
*Когда нужно:* добавляешь новый продукт на сервер или обновляешь существующий.

**Группа 3. Безопасность (периодически).**

**`/audit-security`** — Аудит безопасности. Проходится по чек-листу: правильно ли
настроен SSH, работает ли файрвол, защищены ли `.env`-файлы, не утекли ли секреты
в git-историю. На выходе — отчёт с пометками PASS/WARN/FAIL и рекомендациями. Сам
ничего не чинит — только указывает на проблемы.
*Когда нужно:* раз в квартал, после security-инцидента, перед публикацией нового
сервиса наружу.

**`/rotate-secrets`** — Ротация секретов. Помогает безопасно поменять пароль БД,
API-токен, SSH-ключ или TLS-сертификат: создаёт новый, атомарно подменяет везде где
используется, проверяет что всё работает, удаляет старый.
*Когда нужно:* по расписанию (пароли БД — раз в квартал, API-токены — раз в полгода)
или внепланово при подозрении на компрометацию.

**Группа 4. Сеть и обход блокировок (для оператора в РФ или с серверами в РФ).**

**`/setup-vpn-panel`** — Установка панели управления VPN на твоём сервере. Поднимает
эталонную 3X-UI (это самая популярная панель для VPN), сразу с HTTPS-сертификатом,
случайным логином/паролем (32 символа), нестандартным портом и случайным путём
панели — чтобы её не нашли сканеры. Логин/пароль кладёт в твой менеджер паролей.
Работает для двух сценариев: сервер в РФ (используется как точка входа для семейного
VPN, без маскировки) или сервер за границей (с маскировкой VLESS+Reality для обхода
TSPU).
*Когда нужно:* первый раз — когда хочешь свой VPN-узел.

**`/configure-vpn-routing`** — Настройка собственно VPN на установленной панели.
Создаёт inbound для устройств, добавляет outbound через провайдера VPN
(если у тебя подписка от Blanc/Bebra/похожих) ИЛИ через твой свой заграничный VPS
(если хочешь полный контроль), настраивает маршрутизацию «РФ-сайты напрямую,
остальное через VPN», добавляет балансировщик для нескольких серверов, создаёт
клиентов с UUID.
*Когда нужно:* после `/setup-vpn-panel`, когда добавляешь нового члена семьи в VPN,
когда меняешь провайдера upstream.

**`/setup-server-proxy`** — Серверный прокси для программ. Чтобы бот, скрипт, npm,
pip, curl, git на сервере могли ходить в Anthropic API / OpenAI / GitHub —
программы на сервере не могут пользоваться 3X-UI как Hiddify. Этот скилл создаёт
для них «служебную дверь» на 127.0.0.1:1080, обязательно ставит защиту от
**self-loop** (главный подводный камень — без неё панель падает) и прописывает
переменные окружения с `socks5h://` (без буквы `h` Anthropic возвращает 403).
*Когда нужно:* твой бот падает с «Connection refused» при обращении к API.

**`/generate-client-config`** — Генерация конфигов для устройств. Из существующего
клиента в твоей панели делает: vless://-ссылку, sing-box JSON под платформу
(iPhone / Android / desktop), QR-код для сканирования. Учитывает iOS-боль (старая
версия sing-box внутри клиента) и удаление VPN-приложений из RU App Store —
рекомендует Hiddify или Karing.
*Когда нужно:* добавил нового клиента — нужно прислать ему ссылку или QR.

**Группа 5. Спецоперации (редко, но важно).**

**`/cleanup-existing-server`** — Приведение в порядок сервера, который тебе достался
в наследство. Если на сервере уже есть бардак (контейнеры с непонятными именами,
неправильные права на файлы, дрифт между конфигами nginx) — этот скилл предложит
план чистки и поэтапно выполнит с твоим подтверждением.
*Когда нужно:* если сервер тебе передали с уже работающими сервисами, и там хаос.

**`/migrate-server-to-server`** — Переезд на новый сервер. Если ты сменил провайдера
или хочешь увеличить мощности — этот скилл переносит всю инфраструктуру со старого
сервера на новый: базы данных, файлы, конфигурации. С разными стратегиями —
от «без даунтайма» (сложнее) до «час простоя» (проще).
*Когда нужно:* при смене провайдера или апгрейде.
