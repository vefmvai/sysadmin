<role>
Я извлекаю список VPN-серверов из ЗАКРЫТОЙ подписки провайдера — той, что прячет
сервера за привязкой к устройству (HWID) и/или отдаёт их в нестандартном формате
(массив Xray-профилей). Я работаю для человека, который НИЧЕГО не понимает в
серверах и VPN: сам выбираю технику, сам разбираю формат, а человеку говорю
простым человеческим языком, что происходит и что от него нужно сделать руками.
На выходе — понятный файл со списком серверов и готовыми ссылками.
</role>

<context>
Предполагается:
- У оператора есть ДЕЙСТВУЮЩАЯ платная подписка и ссылка на неё (из личного
  кабинета или бота провайдера). Доступ легальный — это его подписка.
- Провайдер прячет сервера: присылает заглушку (`0.0.0.0` / «App not supported» /
  нулевые UUID) ИЛИ отдаёт Xray-JSON массив профилей вместо vless://-ссылок.
- Для постоянного хранения нужен путь к папке infra (`infrastructure.root_path`).
  Если его нет — сохраню рядом и предупрежу (скилл работает и без полного vpn-блока).

НЕ предполагается:
- Что оператор разбирается в HWID, base64, Reality, форматах подписки. Всё это —
  внутреннее, в логи (`>&2`), НЕ в речь пользователю.
- Что подписка уже заведена в какую-то панель. Это извлечение, не настройка.
- Что у оператора есть свободный слот устройства — это проверяется и при нужде
  он освобождает слот сам в приложении провайдера.
</context>

<goals>
После выполнения должно стать TRUE:
- Определено состояние подписки (живая? зашифрована? сколько трафика/срок).
- Реальный список серверов извлечён (а не заглушка): нормализованный JSON-массив.
- Сервера размечены по странам (из тега/эмодзи-флага; не распознано → «?», не выдумываем).
- Создан читаемый `.txt`: сводка по странам + параметры + готовые vless://-ссылки.
- JSON сохранён в `$INFRA/inventory/shared/vpn-subscriptions/<provider>.json`
  (или рядом + предупреждение, если INFRA не задан).
- Оператор понял простым языком: что получилось (N серверов в таких-то странах),
  где файл, что делать дальше, и что ссылки секретны.
</goals>

# Параметры

| Параметр | Required | Default | Описание |
|---|---|---|---|
| `SUBSCRIPTION_URL` | да | спрошу у оператора (Шаг 1) | Ссылка на подписку из кабинета/бота провайдера |
| `PROVIDER_SLUG` | нет | `subscription` | Короткое имя провайдера для файла (`panterra`, `nurvpn`, `blanc`) |
| `HWID` | нет | добывается `resolve-hwid.sh` | Отпечаток устройства для закрытых подписок. Внутреннее — оператору не показываю |
| `HWID_MODE` | нет | `auto` | `auto` / `happ` (из установленного Happ) / `generate` (свой стабильный) |
| `TXT_PATH` | нет | `$INFRA/.../vpn-subscriptions/<provider>-servers.txt` | Куда положить читаемый файл |

> Все технические параметры — внутренние. Перед оператором их НЕ раскрываю. От
> него прошу только то, что он физически может знать/сделать: дать ссылку,
> посмотреть число устройств в приложении, удалить одно лишнее.

# Процедура

## Шаг 0: Чтение конфига (OPTIONAL)

Скилл работает и без полного vpn-блока — конфиг нужен в основном для `INFRA`
(куда сохранить сервера). Поэтому режим **OPTIONAL**, не STRICT.

```bash
source "$SYSADMIN_ROOT/.claude/skills/_lib/find-config.sh"
find_sysadmin_config optional         # WARN + defaults если нет

INFRA_DIR=""
if [ "$SYSADMIN_CONFIG_FOUND" = "true" ]; then
    RAW_ROOT="$(get_config_field infrastructure.root_path "")"
    [ -n "$RAW_ROOT" ] && INFRA_DIR="$(resolve_infra_path "$RAW_ROOT" "$CONFIG" || true)"
    REPORT_LANGUAGE="$(get_config_field language ru)"
    # Подсказка имени провайдера, если оператор его задавал в vpn-блоке.
    PROVIDER_SLUG="${PROVIDER_SLUG:-$(get_config_field vpn.provider_slug subscription)}"
fi
```

Если `INFRA_DIR` пуст — не падаю: на Шаге 5 сохраню рядом и честно предупрежу.

## Шаг 1: «Дай мне ссылку на подписку» (Green Zone)

Объясняю простым языком, БЕЗ терминов:

> «Чтобы достать твои сервера, мне нужна **ссылка на подписку**. Это длинная
> ссылка, которую тебе выдал провайдер. Где её взять:
> - открой **личный кабинет** провайдера (на сайте) или его **бота в Telegram**;
> - найди кнопку вроде «Скопировать ссылку подписки», «Импорт», «Экспорт
>   конфигурации» или «Subscription URL»;
> - скопируй и вставь сюда.
>
> Не путай с логином-паролем — мне нужна именно длинная ссылка, обычно
> начинается с `https://`.»

Получив ссылку — сохраняю в `SUBSCRIPTION_URL`. Если оператор не знает имя
провайдера — спрошу мягко («как называется сервис? одним словом») для имени
файла; не знает — оставлю `subscription`.

## Шаг 2: Разведка — что это за подписка (Green Zone, READ-ONLY)

Прежде чем что-то тащить, смотрю состояние. Это безопасно — только чтение
заголовков, ничего не меняет.

```bash
SUBSCRIPTION_URL="$SUBSCRIPTION_URL" ./scripts/inspect-subscription.sh > /tmp/verdict.json
KIND="$(jq -r '.kind' /tmp/verdict.json)"
ALIVE="$(jq -r '.alive' /tmp/verdict.json)"
HWID_STATE="$(jq -r '.hwid_state' /tmp/verdict.json)"
TITLE="$(jq -r '.profile_title // ""' /tmp/verdict.json)"
EXPIRE="$(jq -r '.expire_date // ""' /tmp/verdict.json)"
USED="$(jq -r '.traffic_used_gb // ""' /tmp/verdict.json)"
ANNOUNCE="$(jq -r '.announce // ""' /tmp/verdict.json)"
```

Перевожу вердикт в человеческую речь. Примеры:

- `alive=true`, `kind=plain`:
  > «Вижу, подписка живая$([ -n "$EXPIRE" ] && echo ", оплачена до $EXPIRE")$([ -n "$USED" ] && echo ", потрачено $USED ГБ"). Сервера должны отдаться без фокусов — забираю список.»
- `alive=true`, `kind=hwid_locked`:
  > «Подписка живая$([ -n "$EXPIRE" ] && echo " (до $EXPIRE)"). Но провайдер прячет сервера — отдаёт их только твоему устройству, как будто проверяет «свой/чужой». Сейчас аккуратно представлюсь твоим приложением и заберу список — это легально, подписка-то твоя.»
- `alive=false`:
  > «Похоже, срок подписки истёк ($EXPIRE) — провайдер серверов не отдаст. Продли подписку и вернёмся.» (STOP — нечего извлекать.)

Если в `announce` была инструкция/ссылка на бот — запоминаю (пригодится на
Шаге 3, если нужно освобождать слот). Оператору про `x-hwid-*`, base64,
Remnawave — НЕ говорю.

## Шаг 3: Если сервера спрятаны — представляюсь устройством (Yellow Zone)

Только при `kind=hwid_locked`. Логика — в `references/hwid-mechanism.md`.

**3.1 — Сначала пробую без хлопот для оператора.** Добываю «отпечаток» из уже
установленного приложения Happ (если оно есть на маке) — тогда слот НЕ тратится:

```bash
HWID="$(MODE="${HWID_MODE:-auto}" PROVIDER_SLUG="$PROVIDER_SLUG" \
        INFRA_DIR="$INFRA_DIR" ./scripts/resolve-hwid.sh)"
```

Оператору говорю мягко, по-человечески (без слова HWID):

> «Возьму отпечаток твоего устройства из приложения — этого хватит, чтобы
> провайдер меня узнал. Ничего делать не нужно, секунду.»

**3.2 — Если приходит «лимит устройств» (`hwid_state=max_devices_reached`).**
Это значит — все «места под устройства» в подписке заняты. Объясняю просто и
прошу действие, которое человек сделает сам:

> «У тебя в подписке заняты все места под устройства — поэтому новое (моё) не
> влезает. Сделай, пожалуйста, так:
> 1. открой **приложение провайдера** (или его **бота в Telegram**$([ -n "$ANNOUNCE" ] && echo " — вот он: $ANNOUNCE"));
> 2. найди раздел **«Мои устройства»** / **«Управление устройствами»**;
> 3. **удали одно**, которым не пользуешься (например, старый телефон).
>
> Как сделаешь — скажи мне, и я продолжу.»

Это действие НЕ автоматизируется (нет такой кнопки у провайдера через ссылку —
только руками в приложении). Жду ответа оператора, потом повторяю Шаг 3.1.

**3.3 — Если приходит «приложение не поддерживается» (`hwid_state=not_supported`).**
Это лечится автоматически правильным «представлением» — `parse-subscription.sh`
сам пришлёт нужные заголовки. Оператору ничего делать не надо, иду дальше.

## Шаг 4: Извлечение списка серверов (Yellow Zone)

`parse-subscription.sh` сам качает тело, понимает формат (список ссылок /
base64 / Xray-JSON массив профилей / sing-box) и нормализует. Оператор про
форматы НЕ слышит.

```bash
SUBSCRIPTION_URL="$SUBSCRIPTION_URL" HWID="$HWID" \
    ./scripts/parse-subscription.sh > /tmp/servers.json
RC=$?
```

Трактовка кода:
- `0` — успех. Считаю и говорю по-человечески:
  ```bash
  N="$(jq 'length' /tmp/servers.json)"
  # сводка по странам:
  jq -r 'group_by(.country)[] | "  \(.[0].country): \(length)"' /tmp/servers.json
  ```
  > «Готово, нашёл **$N серверов** — вот в каких странах: …»
- `3` — всё ещё заглушка. НЕ сдаюсь молча. Значит: либо слот не освободили, либо
  отпечаток не подошёл. Возвращаюсь к Шагу 3 (сеньор-обёртка):
  > «Провайдер пока не отдаёт список — скорее всего, свободного места под
  > устройство всё ещё нет. Проверь, что удалил одно устройство (Шаг выше), и
  > скажи мне — попробую снова.»
  **Сервера не выдумываю** (правило №1 CLAUDE.md).
- `1` — ни ссылок, ни JSON. Смотрю первые байты ответа (в логе скрипта), решаю:
  возможно Clash YAML (редко) → предложить оператору взять прямой vless-link из
  приложения. Без фантазий.

> 🟡 **Почему это Yellow, а не Green.** Скилл делает реальные сетевые запросы к
> провайдеру под видом фирменного приложения и (по пути B) может занять один
> слот устройства. Это обратимо (слот освобождается в приложении), не
> деструктивно, но оператор должен понимать, что происходит — отсюда брифинг
> на Шагах 2–3.

## Шаг 5: Экспорт «под ключ» — файл + сохранение в infra (Yellow Zone)

```bash
NORMALIZED_JSON=/tmp/servers.json \
PROVIDER_SLUG="$PROVIDER_SLUG" \
INFRA_DIR="$INFRA_DIR" \
${TXT_PATH:+TXT_PATH="$TXT_PATH"} \
./scripts/export-servers.sh > /tmp/txt-path.txt
TXT_FILE="$(cat /tmp/txt-path.txt)"
```

`export-servers.sh` делает:
- читаемый `.txt`: шапка (сколько серверов, когда), сводка по странам,
  по каждой стране — сервера с адресом/транспортом и **готовой vless://-ссылкой**;
- предупреждение про секретность внутри файла;
- JSON в `$INFRA/inventory/shared/vpn-subscriptions/<provider>.json`.

Если `INFRA_DIR` пуст — скрипт сам положит рядом (`./`) и выведет WARN. Тогда
честно скажу оператору, что файл лежит в текущей папке, и предложу запустить
`/sysadmin-init`, чтобы появилось постоянное место.

## Шаг 6: Финальный отчёт «под ключ»

```
✓ Подписка: <title или provider>, живая, до <expire>, потрачено <N> ГБ
✓ Извлечено серверов: <N>
✓ По странам: 🇺🇸 США — X, 🇳🇱 Нидерланды — Y, …
✓ Файл со списком и ссылками: <TXT_FILE>
✓ Сохранено в твоей папке: <INFRA>/inventory/shared/vpn-subscriptions/<provider>.json
```

Дальше говорю простым языком:

> «Вот что получилось: у тебя **$N серверов** в таких странах: … Я сложил их в
> файл — там по каждому серверу есть готовая ссылка, которую можно вставить в
> любое VPN-приложение.
>
> Важно: эти ссылки — как ключи от квартиры. В каждой зашит твой личный код
> доступа. Не выкладывай их в открытые чаты и не пересылай посторонним.
>
> Что можно сделать дальше (по желанию):
> - **завести эти сервера в свою VPN-панель**, чтобы раздавать VPN семье и
>   выбирать страну выхода — это команда `/configure-vpn-routing`;
> - **сделать QR-код и конфиг для телефона/ноутбука** из своей панели —
>   `/generate-client-config`.
>
> Если просто хотел получить список — всё, готово, он в файле.»

# Подводные камни

- **Формат — не только base64.** Panterra/Remnawave отдают **Xray-JSON массив
  профилей** (`[{remarks, outbounds:[...], ...}]`), а не список `vless://`.
  Старый разбор давал ложный «0 серверов». `parse-subscription.sh` проверяет
  `jq empty` и роутит JSON в `parse-xray-json.sh`. См. `references/subscription-formats.md`.
- **`max_devices_reached` ≠ `not_supported`.** Первое лечится освобождением
  слота (действие пользователя в приложении). Второе — правильными заголовками
  Happ (делается автоматически). Не путать диагнозы.
- **HWID оператора из установленного Happ — главный рабочий путь** (путь A):
  устройство уже зарегистрировано, слот не тратим, сервера приходят. Это путь,
  который сработал в боевом прогоне.
- **Освобождение слота делает пользователь** в приложении/боте, НЕ curl.
  Отдельного эндпоинта регистрации/удаления нет (`/register`, `/device` → 502).
- **Гео-блок:** NurVPN на не-РФ IP может отдавать 502/заглушку даже с правильным
  HWID — тогда запрос с РФ-хоста.
- **Ссылки содержат секреты** (рабочий UUID). По умолчанию — в infra (приватный
  репо, в `.gitignore`), НЕ на Desktop, НЕ в git публичного репо, НЕ в открытый чат.
- **Многосерверный профиль `[0]`** (группа одной страны под балансир, теги
  `proxy/proxy-2/...`) разворачивается в отдельные сервера — это норма, не дубль.

# Failed attempts (граблекейс боевого прогона)

- **Трактовать `max_devices_reached` как «тупик — только через бота, обойти
  нельзя»** — НЕВЕРНО (так оступился прошлый прогон: агент сказал «нет способа»
  и встал). Рабочий путь — взять HWID из уже установленного Happ оператора
  (путь A): устройство в списке, заглушки нет, полный конфиг приходит. Бот нужен
  ТОЛЬКО если слот реально надо освободить.
- **Декодировать Xray-JSON как base64** — даёт мусор и ложный «0 серверов».
  Тело Xray-подписки — это JSON, не base64. Сначала `jq empty`, потом — решение
  о декодировании.
- **Заводить все извлечённые сервера скопом в один балансир** — это уже задача
  `/configure-vpn-routing` (и там есть защита от «скачущего IP по странам»).
  Здесь — только извлечение и сохранение, без маршрутизации.
