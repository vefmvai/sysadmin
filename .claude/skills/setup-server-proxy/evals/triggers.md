# Evals для /setup-server-proxy

> Источник правды для `description` в SKILL.md.

## ДОЛЖНО триггерить (positive)

- «настрой прокси на сервере»
- «бот не может в Claude API»
- «бот не видит api.anthropic.com»
- «npm не работает на сервере»
- «pip не качает пакеты»
- «дай серверу доступ к заблокированным API»
- «hugo/curl не идёт за границу»
- «настрой прокси через мою VPN-панель для программ на сервере»
- «нужен серверный прокси чтобы бот ходил через VPN»
- «server proxy через VPN panel»
- «бот возвращает 403 от Anthropic»
- «git push не работает с сервера в github»

## НЕ ДОЛЖНО триггерить (negative)

- «настрой VPN-панель» — это `/setup-vpn-panel`
- «добавь клиента VPN» — это `/configure-vpn-routing` (клиенты для устройств, не серверного прокси)
- «настрой VLESS+Reality на клиенте» — это клиент, `/generate-client-config`
- «расскажи что такое socks5h» — это вопрос про знания, ответ персоны со ссылкой на references
- «открой порт 1080 на сервере» — частичное действие, скилл делает namek намного больше
- «удали серверный прокси» — обратная операция, скилл этого не делает
- «настрой Privoxy чтобы npm видел SOCKS» — частный кейс с другим инструментом, не покрывается основной процедурой скилла → ответ персоны с пояснением (см. socks5-vs-socks5h.md «А если программа не поддерживает SOCKS5?»)
- «aiohttp не работает через прокси» — это уже после установки прокси, troubleshooting → персона со ссылкой на python-libs-with-proxy.md

## Edge-cases

- «настрой прокси» (без контекста) — должен уточнить «для программ на сервере или для устройств?». Сервер → этот скилл, устройства → `/generate-client-config`.
- «бот падает с какой-то ошибкой при обращении к API» — уточнить, в чём симптом. Если «Connection refused» / «Could not resolve» — этот скилл. Если 403 / authorization — другие причины, не VPN.
- «у меня уже есть прокси на сервере, не через 3X-UI» — скилл сходит с дистанции: «найди мою предыдущую настройку и я подскажу как заменить», но автоматизированно не пересоздаёт.
- «бот ходит в локальный API на той же машине» — не нужен VPN, нужен `no_proxy=localhost,127.0.0.1`. Скилл это включает в /etc/environment по умолчанию.
- «aiohttp игнорирует HTTPS_PROXY» — это troubleshooting после скилла → ответ персоны со ссылкой на `python-libs-with-proxy.md` (нужно `trust_env=True`).
- «hugo не работает на сервере» — Go-программа. Go stdlib не уважает HTTPS_PROXY для SOCKS5 → нужен Privoxy. Скилл предупреждает в финальном отчёте.
