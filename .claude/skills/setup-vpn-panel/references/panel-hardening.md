# Безопасность панели 3X-UI: чек-лист

Скилл `/setup-vpn-panel` применяет 8 защитных мер. Этот документ — обоснование
каждой меры и инструкции для аудита (как проверить, что мера сработала).

## 1. Smart-логин (≠ `admin`)

**Что делается:** установщик создаёт случайные креды, но при установке через
интерактивный диалог можно случайно оставить `admin/admin` (Docker-вариант
имеет жёсткий дефолт). Скилл **сразу после установки** меняет логин на
сгенерированный 8-символьный (буквы + цифры).

**Зачем:** `admin` — первое имя, которое перебирают брутфорсеры. Случайный
логин даёт **дополнительный 1-й уровень секрета** (классический парадокс —
не «security through obscurity», а реальное расширение пространства
поиска).

**Аудит:** `ssh $TARGET '/usr/local/x-ui/x-ui setting -show true | grep -i user'`
— значение не должно быть `admin`.

## 2. 32-символьный пароль

**Что делается:** скрипт `02-install-3x-ui.sh` генерирует пароль из 32
случайных символов через `/dev/urandom` (`api_random_string` в lib-api).

**Зачем:** при брутфорсе через WAF/панель ~3 попытки/сек = 32^32 паролей.
Достаточно для защиты до конца времён.

**Аудит:** длина пароля = 32, нет в утечках (`haveibeenpwned`).

## 3. Нестандартный порт панели

**Что делается:** установщик ставит порт по умолчанию 2053 (Docker) или
случайный (host-install). Скилл меняет на пользовательский PANEL_PORT
из диапазона 1024-65535 (рекомендуется 20000-60000 — не сканируются массово).

**Зачем:** автоматические сканеры (Shodan, Censys) индексируют типовые
порты (2053, 54321 — стандарт x-ui). Нестандартный = меньше шума в логах
(не «security through obscurity», но снижает атаки-discovery).

**Аудит:** `ss -tlnp | grep ':$PANEL_PORT'` — слушается. Дефолтные порты
(2053, 54321) свободны.

## 4. Случайный webBasePath

**Что делается:** скилл генерирует webBasePath из 10 случайных символов
(`abc123xyz0`) и применяет через `x-ui setting -webBasePath`.

**Зачем:** даже если злоумышленник нашёл порт сканированием, он попадает
на 404 без webBasePath. Это **второй уровень секрета** до экрана логина.

**Аудит:** `curl -sI https://$DOMAIN:$PANEL_PORT/` → 404 (или 200 без формы).
`curl -sI https://$DOMAIN:$PANEL_PORT/$WEB_BASE_PATH/` → 200 с формой логина.

## 5. HTTPS обязательно (Let's Encrypt)

**Что делается:** скрипт `03-configure-tls.sh` выпускает Let's Encrypt
сертификат, привязывает к панели. Без HTTPS пароли админа уходят
plaintext по сети.

**Зачем:** базовая гигиена. Auto-renew через acme.sh hook
(`--reloadcmd "systemctl restart x-ui"`).

**Аудит:** `curl -sI https://$DOMAIN:$PANEL_PORT/$WEB_BASE_PATH/` →
HTTP/2 200 с валидным сертификатом. `openssl s_client -servername $DOMAIN
-connect $DOMAIN:$PANEL_PORT -tls1_3 < /dev/null` → cert chain валиден.

## 6. UFW: только нужные порты

**Что делается:** `04-ufw-setup.sh` открывает:
- 22/tcp — SSH (всегда).
- PANEL_PORT/tcp — панель.
- 443/tcp — только при `LOCATION=foreign-server` (VLESS+Reality inbound).
- 80/tcp — **закрыт** после выпуска cert (acme renew открывает временно).

**Зачем:** минимальная attack surface. Default deny incoming.

**Аудит:** `ssh $TARGET 'ufw status verbose'` — Status: active, default
deny incoming, открыты только перечисленные порты.

## 7. fail2ban для панели

**Что делается:** **не делается этим скиллом**. Базовый fail2ban для SSH
устанавливается `/bootstrap-new-server`. fail2ban для самой панели 3X-UI
(защита от брутфорса логина) — отдельная задача, не закрывается этим скиллом.

**Открытая работа:** добавить опциональный шаг «fail2ban jail для 3X-UI»
в виде todo. Конфигурация jail должна парсить логи панели на predicate
«failed login» и блокировать IP на 1 час после 5 попыток.

**Аудит:** `ssh $TARGET 'fail2ban-client status'` — показывает активные jails.
Jail для x-ui — нет (текущее состояние).

## 8. Сохранение секретов в менеджер паролей

**Что делается:** `05-record-secrets.sh` записывает в menager оператора:
- URL панели (полный с webBasePath).
- Логин администратора.
- Пароль (32 символа).
- Notes с alias сервера.

Менеджер выбирается из `sysadmin-config.json` (`secrets.manager`).

**Зачем:** секреты в git = катастрофа. Все секреты у оператора в одном
проверенном месте, бэкап менеджера = бэкап секретов.

**Аудит:**
- macOS: `security find-generic-password -s "3xui-panel-$SERVER_ALIAS" -w`.
- pass: `pass show "3xui-panel-$SERVER_ALIAS"`.
- bitwarden: `bw get item "3xui-panel-$SERVER_ALIAS"`.
- 1Password: `op item get "3xui-panel-$SERVER_ALIAS"`.

## Что НЕ делается этим скиллом (явно)

- **fail2ban для панели** — отдельная задача (см. §7).
- **WAF (Web Application Firewall) перед панелью** — например, Caddy с
  rate-limit или nginx с `limit_req_zone`. Можно добавить позже.
- **2FA для панели** — 3X-UI поддерживает 2FA через TOTP (см. `x-ui setting
  -resetTwoFactor true/false`), но скилл его не настраивает. Открытая работа.
- **IP-allowlist для админ-доступа** — например, через UFW `allow from $YOUR_IP
  to any port $PANEL_PORT`. Не реализовано, потому что у оператора может
  меняться IP (мобильная сеть, разные локации). Можно делать опциональным
  параметром.
- **Изоляция Telegram-токена** — Telegram-бот (опциональный, через
  `/install-monitoring-stack` или вручную) живёт в той же панели. Скомпрометированный
  токен Telegram-бота даёт доступ к chat ID, но не к самой панели.

## Итоговый чек-лист

После запуска `/setup-vpn-panel` — пройти ручной аудит:

- [ ] Логин ≠ `admin`
- [ ] Пароль 32 символа в менеджере паролей
- [ ] Порт ≠ 2053/54321 (стандартные)
- [ ] webBasePath случайный (≥6 символов)
- [ ] HTTPS с Let's Encrypt сертификатом, замок зелёный в браузере
- [ ] UFW: deny incoming + только PANEL_PORT/443/22 открыты
- [ ] Секреты в менеджере паролей, не в git
- [ ] (Опц.) fail2ban-jail для самой панели
